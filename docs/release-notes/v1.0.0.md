# v1.0.0 — Product as shipped

**Android GA Tracking Debugger** is a **desktop** tool (Python 3 + Tkinter) for **local** measurement debugging. It does **not** talk to Google’s servers: it reads **`adb logcat`**, finds Firebase / Google Analytics **debug** lines, and shows them in one place in near real time.

## What you can do in this release

| Area | Behavior |
|------|----------|
| **Events** | Lines that look like `Logging event` with name and parameters. |
| **User properties** | `Setting user property` and the `Setting user property (FE):` variant. |
| **Consent** | `Setting storage consent` (with change history for export). |
| **Multi-device** | If `adb devices` lists more than one, pick the serial; all logcat calls use `adb -s`. |
| **Session** | Status line (ADB / device / capturing), search, **JSON export** of the session, **double-click** for row detail. |
| **Languages** | English and Spanish from the **Languages** menu. |

## Requirements

- [Android platform-tools](https://developer.android.com/tools/releases/platform-tools) (`adb`) on your `PATH`
- A phone or emulator with USB or wireless debugging
- **Python 3.7+** to run `python main.py` (stdlib only for the app; `pip install -r requirements-dev.txt` for contributors who run tests)

## What this is *not*

- Not a replacement for **Firebase / GA4** web reports, **DebugView** in the cloud, or **BigQuery** exports.
- It answers: *“What is the Analytics SDK printing in logcat on this device?”* — not *“Did the event hit Google’s backend?”*
